ASSET PATH

素材路径维护 / Asset Path Manager

GitHub Pages 静态站不能在线写入仓库。正式素材请先上传到 GitHub 仓库 assets 目录，再把路径写入 assets/data/site-data.json。此处保存只写入当前浏览器 localStorage，用于本机预览。

推荐目录：assets/projects/项目ID-英文短名/cover.jpg、detail-hero.jpg、article-cover.jpg、gallery-01.jpg、walkthrough.mp4、video-poster.jpg、model.glb、model-thumb.jpg、panorama.jpg、panorama-thumb.jpg、case.pdf。多图字段请一行一个路径；附件格式为：file path | file name | file type | description。大小写必须与 GitHub 仓库文件完全一致。

本机上传仅用于预览。若要让所有设备看到，请将素材文件上传到 GitHub 仓库对应 assets 目录，并在后台填写项目内相对路径。

SECTION BG

板块背景 / Section Backgrounds

板块背景以路径方式维护。请先把背景图提交到 GitHub 仓库 assets/backgrounds 目录，再填写相对路径；未填写时保持当前默认效果。首页 #home.hero 也会使用这里的背景路径。

本机上传仅用于预览。若要让所有设备看到，请将素材文件上传到 GitHub 仓库对应 assets 目录，并在后台填写项目内相对路径。

ARTICLE

项目文章 / Article

为完整项目页维护图文长文。图片、GIF、视频、PDF、模型和全景都使用 assets 相对路径；gallery 请一行一个路径。

本机上传仅用于预览。若要让所有设备看到，请将素材文件上传到 GitHub 仓库对应 assets 目录，并在后台填写项目内相对路径。

PUBLISH CHECK

素材路径检查 / Publish Checklist

检查当前后台正在使用的数据。它会列出空路径、本机路径、可疑路径，以及导出发布时所有被引用的素材路径。

所有正式素材都应使用项目内相对路径，例如 assets/projects/project-a/cover.jpg。不要使用 C:\Users\...、/Users/... 或 file://...。
